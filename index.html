Objectif & Contraintes

Nous avons choisi le sujet Vidéo à la demande. Notre projet respecte les contraintes suivantes :

  • Utilisation de l'image de base debian:latest.
  • Un seul service réseau déployé : un serveur Web Nginx.
  • Aucun outil multi-conteneurs (pas de docker-compose).

Architecture du projet

Notre projet repose sur une architecture simple basée sur un seul conteneur Docker.

  • Un client (navigateur web)
  • Un conteneur Docker
  • Un serveur web Nginx
  • Des fichiers vidéo servis en HTTP

Le client envoie une requête au serveur via un port exposé (8080), qui est redirigé vers le port 80 du conteneur Docker où Nginx est actif.

Cette architecture permet de simplifier le déploiement tout en respectant les contraintes du projet.

Docker (Fonctionnement)

Pour le projet nous avons utilisé Docker. Il fonctionne ainsi :

  • Création de l’image : une image Docker est créée à partir d’un fichier appelé Dockerfile. Dans ce fichier, on décrit étape par étape comment construire l’environnement : par exemple le système de base (comme Debian ou Alpine), l’installation des dépendances, et l’ajout de notre application. Une fois construite, l’image est immuable (elle ne change plus) et peut être réutilisée autant de fois que nécessaire.
  • Lancement du conteneur : un conteneur est une instance d’une image Docker en cours d’exécution. On peut voir ça comme une “copie active” de l’image. C’est dans ce conteneur que l’application tourne réellement. On peut démarrer, arrêter ou supprimer un conteneur très rapidement, ce qui rend les déploiements simples et rapides.
  • Isolation : chaque conteneur fonctionne dans son propre environnement isolé. Cela signifie qu’il possède ses propres fichiers, ses propres dépendances et ses propres configurations. Ainsi, deux applications peuvent utiliser des versions différentes d’un même outil sans entrer en conflit. Cela évite aussi de “casser” le système principal.
  • Portabilité : une fois l’image créée, elle peut être exécutée exactement de la même manière sur n’importe quel environnement : ordinateur personnel, serveur ou cloud. Docker garantit que l’application fonctionnera de façon identique, peu importe le système d’exploitation (Linux comme Debian ou Alpine, mais aussi Windows ou Mac via Docker). Cela facilite énormément le partage et le déploiement des applications.
  • Exemple concret : imaginons qu’un nouveau développeur arrive dans l’équipe avec un Mac, alors que les autres travaillent sur Windows ou Linux. Sans Docker, il devrait installer toutes les dépendances, gérer les versions, et pourrait rencontrer des erreurs différentes des autres. Avec Docker, il lui suffit de récupérer le projet et de lancer les conteneurs : l’environnement est déjà configuré exactement comme pour les autres développeurs. Résultat : il peut travailler immédiatement, sans problème de compatibilité.

Quelques commandes Docker utiles

  • docker info
    Fourni des informations sur docker et la machine utilisée
  • docker run \image\
    On crée un conteneur à partir d'une image. Si nous n'avons pas l'image localement, elle est téléchargé automatiquement
  • docker stop|start \id\
    Arrêter | Lancer un conteneur
  • docker ps -a
    Liste des conteneurs (-a affiche également les conteneurs arrêtés)
  • docker rm -f \id\
    Forcer la suppression d un conteneur même s'il n'est pas arrêté
  • docker exec -it \id\ bash
    Ouvrir un terminal interactive (-it) dans le conteneur

Les \id\ ou \image\ correspond a l'id d'un conteneur ou au nom d'une image

Analyse du Dockerfile

Voici le fichier utilisé pour construire notre image Docker :

FROM debian:latest

RUN apt-get update && apt-get install -y nginx wget \
&& rm -rf /var/lib/apt/lists/*

COPY liens_videos.txt /liens_videos.txt
COPY entrypoint.sh /entrypoint.sh

RUN chmod +x /entrypoint.sh

EXPOSE 80

CMD ["/entrypoint.sh"]

Explication :

  • Debian : système de base léger et stable
  • Nginx : serveur web pour servir les vidéos
  • wget : permet de télécharger les vidéos automatiquement
  • COPY : ajoute les fichiers nécessaires au conteneur
  • EXPOSE 80 : ouverture du port HTTP
  • CMD : lance le script au démarrage

Script de démarrage (entrypoint.sh)

Au lancement du conteneur, un script est exécuté automatiquement.

  • Lit une liste de liens vidéo
  • Télécharge les vidéos avec wget
  • Place les fichiers dans le dossier web de Nginx
  • Lance ensuite le serveur Nginx

Ce fonctionnement permet de rendre le conteneur dynamique : les vidéos ne sont pas stockées dans l'image mais récupérées au lancement.

Démonstration (Installation)

Pour lancer notre service sur votre machine, exécutez les commandes suivantes :

1. Construire l'image (Installation de Nginx et copie des médias) :

docker build -t debianflix .

2. Lancer le conteneur (Mappage du port 80 vers 8080) :

docker run -d -p 8080:80 debianflix

3. Accéder au service web : http://localhost:8080
